30 ## Example Usage {#sec-example-usage}
31 
32 Run a Drush Rush command like:
33 
34  drush rush myjob.init
35 
36 To parse the job folder's rush files for build parameters and build operations and then execute the following operations:
37 
38 1. Make the build directory
39 2. Put the directory in the web root
40 3. Add other directories and files to the build (setup the build skeleton)
41 4. Create a docroot folder for the Drupal code base
42 5. Build the code base with Drush Make using specified makefile(s) and prepare the site for install
43 6. Create the specified database
44 7. Create the specified dns and virtual host entries
45 8. Run any pre-build commands
46 9. Run any pre-build scripts
47 10. Create build aliases.drushrc.php files
48 11. Install the site
49 12. Change variable values as specified
50 13. Run any post-build commands
51 14. Run any post-build scripts
52 15. Put the build under version control
53 16. Create a remote repo (such as on GITHUB), commit the code, and push it

71 ## Background {#sec-background}
72 
73 This project originates out of experience using Jenkins to automate building and testing Drupal projects.
74 
75 > **Note:** Jenkins is an extendable open source continuous integration server. Visit: http://jenkins-ci.org/ for more information.
76 
77 Beyond its excellent tool set to execute and monitor the results of repeated build jobs, Jenkins is a good tool to automate repetitive workflows for developers.
78 
79 For example, if a developer needs to work on a website locally, he or she typically must:
80 
81 1. Get the code via a Versions Control System like git
82 2. Switch to a branch or tag to work on
83 3. Change or add some parameters in settings files to match the development environment
84 4. Get the db export
85 5. Create a testing db
86 6. Import the db snapshot
87 7. Create a local host file entry
88 8. Restart Apache
89 
90 That's eight steps before starting to code.
91 
92 Jenkins can automate many of these common developer workflows, using job parameters to let developers simply enter a project name and a branch and then run a Jenkins job to do their repetitive build work for them.
93 
94 Like Jenkins, Drush Rush is a tool to run parameterized builds with Drush.
95 In most cases the build jobs are a series of core drush commands chained together to complete a full build process with a single command.
96 
97 ### A Few Points about Drush Rush Jobs
98 
99 1. Drush Rush jobs are **written once then used repeatedly**. *There is probably little value to writing a Drush Rush job for a one time build process.*
100 2. Drush Rush jobs are **extendable**. Complex jobs may be written by extending and or combining similar jobs.
101 3. Drush Rush jobs are **shareable** so a job written by one developer may be shared with many.
102 4. Drush Rush jobs are **light weight**. Drush Rush may be used to execute steps as part of a continuous integration or automated testing process, however, it is not a CI or Testing Suite.
