14 ## Jobs and Builds {#sec-builders-jobs-builds}
15 
16 *Note:* **The following is an extended version of the Jobs and Builds section for Rush users. This version has more information specifically for Rush Job builders. **
17 
18 The two main components of Drush Rush are **jobs** and **builds**.
19 
20 Jobs define what to produce and builds are what is produced.
21 
22 Jobs define parameters and operations used to execute a build (typically installing a Drupal website).
23 
24 Conversely, builds are the product of the execution of a job.
25 
26 Jobs, once written rarely change. Builds are created and deleted often.
27 
28 Developers share job folders to give others the ability to create the build defined by the job.
29 
30 To recap:
31 
32 - Jobs
33  - A package of parameters and operations, which when run via the drush rush command will produce a build.
34 
35 - Builds
36  - The product of a job; typically a Drupal website, though a build may be something as simple as adding a domain to a hosts file.
37 
38 - Parameters
39  - Defined in params.ini or params.php files, parameters provide variables for a build, such as site name, repo information, database connection strings, and really anything needed to complete a build.
40 
41 - Operations
42  - Operations run code commands (often in Drush, shell, php, git, and mysql) to act upon the file system, interact with other systems, and to build Drupal websites.
43 
44 ## File Types {#sec-file-types}
45 
46 - Parameter Files
47  - Contain environmental, application, and job based variables to use at run time to build a project.
48  - Parameter files come in two flavors: `params.ini` and `params.php`
49  - Most jobs use `params.ini` files. `params.php` files give you the option to use business logic when setting rush params.
50 
51 - Rush Files
52  - Contain a list of operations to complete a build.
53  - Rush files come in two flavors: `rush.ini` and `rush.php`
54  - Most jobs use `rush.ini` files. `rush.php` files give you the option to use business logic when running drush operations.
55  - Drush Rush will default to running rush files with the default names of 'rush.ini' or 'rush.php'.
56  - You may specify a different rush file to run at runtime by adding the `--rush=<rush_file>` parameter to your drush rush command.

60 ## Rush Application Folder {#sec-builders-application-folder}
61 
62 *Note:* **The following is an extended version of the Rush Application Folder section for Rush users. This version has more information specifically for Rush Job builders. **
63 
64 - Rush (`~/Rush/`)
65  - The Drush Rush application folder in your HOME directory, which contains an `environment` directory and a `jobs` directory.
66  - This directory is for user environment settings, default settings for all jobs, and default make files to be used by Drush Make.
67 
68 - environment (`~/Rush/environment`)
69  - Contains one or more params files and may also contain a global make file.
70  - Put parameters to use as defaults for most jobs. Then override these parameters at the job level if needed.
71 
72 - jobs (`~/Rush/jobs`)
73  - Contains job folders.
74 
75 - a job folder (`~/Rush/jobs/jobName`)
76  - Contains a rush file with operations and typically a params file for the specific job.
77  - A job folder may also contain a default.make file.
78  - A job folder may also contain a sub job folder.
79 
80 - a sub job folder (`~/Rush/jobs/jobName/subJobName`)
81  - A sub job folder will inherit the parameters of its parent job.
82  - A sub job may override a parameter from its parent job.
83  - A sub job without a rush file will run the rush operations from its parent, while using its own parameters.
84  - By design, a sub job with its own rush operations will still inherit parameters from its parent, but not the rush operations from its parent.
85  - Instead, the sub job rush operations file may call the parent job operations to run at any point within its own operations, by using the command `[j] = parentJobName`

150 #### More About Operations {#sec-commands-operations-ops-more}
151 
152 Operations map to functions defined in the `operations` directory of the Drush Rush codebase.
153 
154 Groups of operation functions include:
155 
156 - **File** system functions for file and directory management.
157 - **Host** functions for managing dns and vhost entries as well as Apache.
158 - **SQL** functions for database management.
159 - **Site** functions for creating, deleting, and managing Drupal site installs.
160 - **Repo** functions to manage version control for code.
