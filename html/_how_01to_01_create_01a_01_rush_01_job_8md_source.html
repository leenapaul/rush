11 ## 1. Identify the Goal of the Build and the Steps to Accomplish the Goal {#sec-goal}
12 
13 A good start for using Drush Rush is to identify what you would like accomplish and what the steps are to accomplish your goal.
14 
15 Let's say you want to start a new Drupal website and put it under version control before you start working on it.
16 
17 Let's call this goal "Init Site".
18 
19 The steps to accomplish your "Init Site" goal might include:
20 
21 1. Run `drush make` to build your codebase. (Or download and unzip the codebase and each module you want to start with.)
22 2. Go to the directory of the codebase `drush make` built.
23 3. Run `drush site-install` to install the Drupal site.
24 4. Run `git init` to put the build under version control.
25 
26 Now these steps are not hard to do and are not that many.
27 But what if you find yourself doing these steps more than a few times a week?
28 What if you could shrink these steps down to one step and then not have to think about the other steps again?
29 
30 Let's turn the tasks for the goal "Init Site" into a single command called:
31 
32  drush rush InitSite
33 
34 In the next section we will create a Drush Rush job for "Init Site" goal.
35 
36 ## 2. Write the Drush Rush Job {#sec-write}
37 
38 ### The General Steps to Create a Drush Rush Job
39 
40 There are three steps involved in creating any new Drush Rush job:
41 
42 1. Create a job folder and give it a job name.
43 2. Write a rush file with operations that match the steps needed to accomplish your goal.
44 3. Optionally, write a params file to add custom variables to your build job.
45 
46 ### Let's Write the InitSite Job
47 
48 #### 1. Create the Job Folder
49 
50 For a typical Drush Rush setup, the Rush Application jobs folder will be located here `~/Rush/jobs` .
51 
52 Make a directory in the Rush jobs folder and name it: `InitSite`
53 
54 **Note:** The name of a Rush job directory may not contain spaces or a period.
55 
56 #### Write the Rush File
57 
58 See the section titled, 'File Types' on the [[Key Terminology|Key Terminology]] page.
59 Add a text file named `rush.ini` to the `InitSite` job folder.
60 
61 The rush file should be located here: `~/Rush/jobs/InitSite/rush.ini`
62 
63 Open the rush file in a text editor and add the following text:
64 
65  [f] = create_build_directory
66  [f] = make
67  [f] = si
68  [f] = git_init
69 
70 Each of the lines above defines a command for Drush Rush to run when building your job.
71 
72 The 'f' in the syntax `[f]` tells Drush Rush to execute a function.
73 
74 In the first line above, the syntax `[f] = create_build_directory` tells Drush Rush to execute a function which the `create_build_directory` operation maps to.
75 
76 The syntax `[f] = make` tells Drush Rush to go to the build directory and run Drush make. Drush Rush will look for a make file specified in your job's params.ini file if you have one, a make file within your job folder if you have one, and then in your `environment` directory.
77 
78 The syntax `[f] = si` tells Drush Rush to go to the directory of the codebase that `drush make` just built and then run `drush site-install` to install the Drupal site.
79 
80 Lastly, the syntax `[f] = git_init` tells Drush Rush to go to the top level of the build directory and initialize a git repository to put the build under version control.
81

84 ## 3. Check the Job {#sec-check}
85 
86 This step is optional. It defines a few ways to inspect your job's parameters and operations before you run it.
87 
88 The following command lists the Drush Rush jobs within your Rush Application folder.
89 
90  drush rush-list
91 
92 When you run it you should see something like this:
93 
94  **********JOBS LIST********************************************
95 
96  InitSite
97 
98  **********END LIST********************************************
99 
100 The output should contain the name of your new job.
101 
102 If you do not see your job in the list see {#sec-create-app-directory} and {#sec-application-folder} for information about setting up your Rush Application folder.
103 
104 Run the following command to see the list of operations your job will execute when run.
105 
106  drush rush InitSite --show=ops
107 
108 You should see the following output:
109 
110  ********** OPERATIONS ********************************************
111 
112  Array
113  (
114  [0] => Array
115  (
116  [f] => create_build_directory
117  )
118 
119  [1] => Array
120  (
121  [f] => make
122  )
123 
124  [2] => Array
125  (
126  [f] => si
127  )
128 
129  [3] => Array
130  (
131  [f] => git_init
132  )
133 
134  )
135 
136 
137  ********** END OPERATIONS ********************************************
138 
139 The operations listed in the array output will match the operations within your `rush.ini` file.
