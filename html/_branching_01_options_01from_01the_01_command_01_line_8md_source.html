8 Branching Options from the Command Line {#page-branching-options-from-command-line}
9 ====================
10 
11 We have three new options which can be used when calling the `drush rush` command.
12 
13 Let's start with the difference between the last option and the other two.
14 
15 ### `--create-branch`
16 
17 The `--create-branch` is the only drush rush option which will pass a parameter
18 from the command line **AND** call a rush operation.
19 
20 Passing `--create-branch`will call the drush operation `git_create_branch` at the
21  **end** of the run of a rush job.
22 
23 Here are a few points about using this parameter at runtime:
24 
25 * `git_create_branch` basically runs `git checkout -b`
26 * it does not depend on the `create_branch` operation being present in the job
27 rush.ini file
28 * whether the parameter is present in the params.ini file or not, the `create_branch`
29 operation will always run **after** the full rush job has completed
30 
31 This command parameter is the most useful and truly new option.
32 
33 ### `--branch`
34 
35 The `--branch` parameter is used with the `git_clone` operation. Using this
36 parameter `--branch=otherbranch` from the command line will tell `git_clone` to
37 clone the branch named 'otherbranch' rather than what is specified in the
38 params.ini file. Thus `--branch` at the command line is simply an override for
39 the `[repo][branch]` parameter set in a rush job's params.ini file.
40 
41 The following are a few scenarios which can occur with this param and operation:
42 
43 1. No `[repo][branch]` parameter has been set in the params.ini file =
44 The `git_clone` operation will clone the `master` branch when the operation is
45 run in a rush.ini file.
46 2. No `[repo][branch]` parameter has been set in the params.ini file AND `--branch`
47 has been set to 'otherbranch' at runtime = The `git_clone` operation will clone
48 the already existing branch named 'otherbranch' when the operation is run in a
49 rush.ini file.
50 3. The parameter `[repo][branch]` has been set in the params.ini file to 'develop'
51 = The `git_clone` operation will clone the `develop` branch when the operation
52 is run in a rush.ini file.
53 4. The parameter `[repo][branch]` has been set in the params.ini file to
54 'develop' AND `--branch` has been set to 'master' at runtime = The `git_clone`
55 operation will clone the `master` branch when the operation is run in a rush.ini file.
56 
57 ### `--new-branch`
58 
59 The `--new-branch` will likely become less useful now that we can create a new
60 branch from the command line call to the job. Before, this parameter was used
61 with the rush operations `git_branch` and `git checkout`.
62 
63 Unlike the `--create-branch` parameter, this parameter at runtime only sets or
64 overrides a parameter normally set in params.ini. The new parameter will be used,
65 when one or both rush operations `git_branch` and `git checkout` are called in
66 the rush.ini file.
