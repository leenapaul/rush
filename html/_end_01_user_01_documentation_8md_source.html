33 ## Tutorial - How to Create a Drush Rush Job {#sec-tutorial}
34 
35 ### 1. Identify the Goal of the Build and the Steps to Accomplish the Goal {#sec-quick-start-1}
36 
37 A good start for using Drush Rush is to identify what you would like accomplish and what the steps are to accomplish your goal.
38 
39 Let's say you want to start a new Drupal website and put it under version control before you start working on it.
40 
41 Let's call this goal "Init Site".
42 
43 The steps to accomplish your "Init Site" goal might include:
44 
45 1. Run `drush make` to build your codebase. (Or download and unzip the codebase and each module you want to start with.)
46 2. Go to the directory of the codebase `drush make` built.
47 3. Run `drush site-install` to install the Drupal site.
48 4. Run `git init` to put the build under version control.
49 
50 Now these steps are not hard to do and are not that many.
51 But what if you find yourself doing these steps more than a few times a week?
52 What if you could shrink these steps down to one step and then not have to think about the other steps again?
53 
54 Let's turn the tasks for the goal "Init Site" into a single command called:
55 
56  drush rush InitSite
57 
58 In the next section we will create Drush Rush job for "Init Site" goal.
59 
60 ### 2. Write the Drush Rush Job {#sec-tutorial-2}
61 
62 #### The General Steps to Create a Drush Rush Job {#sec-tutorial-2a}
63 
64 Before we create the InitSite job, here are the three steps involved in creating any new Drush Rush job:
65 
66 1. Create a job folder and give it a job name.
67 2. Write a rush file with operations that match the steps needed to accomplish your goal.
68 3. Optionally, write a params file to add custom variables to your build job.
69 
70 #### Let's Write the InitSite Job {#sec-tutorial-2b}
71 
72 ##### Create the Job Folder
73 
74 For a typical Drush Rush setup, the Rush Application jobs folder will be located here `~/Rush/jobs` .
75 
76  - *See @ref sec-create-app-directory and @ref sec-application-folder *
77 
78 Make a directory in the Rush jobs folder and name it: `InitSite`
79 
80 **Note:** The name of a Rush job directory may not contain spaces or a period.
81 
82 ##### Write the Rush File
83 
84 See @ref sec-file-types for a definition of a Rush File.
85 
86 Add a text file named `rush.ini` to the `InitSite` job folder.
87 
88 The rush file should be located here: `~/Rush/jobs/InitSite/rush.ini`
89 
90 Open the rush file in a text editor and add the following text:
91 
92  [f] = create_build_directory
93  [f] = make
94  [f] = si
95  [f] = git_init
96 
97 Each of the lines above define a command for Drush Rush to run when building your job.
98 
99 The 'f' in the syntax `[f]` tells Drush Rush to execute a function.
100 
101 In the first line above, the syntax `[f] = create_build_directory` tells Drush Rush to execute a function which the `create_build_directory` operation maps to.
102 
103 The syntax `[f] = make` tells Drush Rush to go to the build directory and run Drush make. Drush Rush will look for a make file specified in your job's params.ini file if you have one, a make file within your job folder if you have one, and then in your `environment` directory. See {#sec-application-folder} for information about the `environment` directory.
104 
105 The syntax `[f] = si` tells Drush Rush to go to the directory of the codebase that `drush make` just built and then run `drush site-install` to install the Drupal site.
106 
107 Lastly, the syntax `[f] = git_init` tells Drush Rush to go to the top level of the build directory and initialize a git repository to put the build under version control.
108 
109 See @ref sec-operations for more about Rush Operations.
110 
111 ### 3. Check the Job
112 
113 This step is optional. It defines a few ways to inspect your job's parameters and operations before you run it.
114 
115 The following command lists the Drush Rush jobs within your Rush Application folder.
116 
117  drush rush-list
118 
119 When you run it you should see something like this:
120 
121  **********JOBS LIST********************************************
122 
123  InitSite
124 
125  **********END LIST********************************************
126 
127 The output should contain the name of your new job.
128 
129 If you do not see your job in the list see {#sec-create-app-directory} and {#sec-application-folder} for information about setting up your Rush Application folder.
130 
131 Run the following command to see the list of operations your job will execute when run.
132 
133  drush rush InitSite --show=ops
134 
135 You should see the following output:
136 
137  ********** OPERATIONS ********************************************
138 
139  Array
140  (
141  [0] => Array
142  (
143  [f] => create_build_directory
144  )
145 
146  [1] => Array
147  (
148  [f] => make
149  )
150 
151  [2] => Array
152  (
153  [f] => si
154  )
155 
156  [3] => Array
157  (
158  [f] => git_init
159  )
160 
161  )
162 
163 
164  ********** END OPERATIONS ********************************************
165 
166 The operations listed in the array output will match the operations within your `rush.ini` file.
